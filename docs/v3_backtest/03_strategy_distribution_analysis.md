## 3.3 事件驱动策略分析

### 为什么 V2 只有 8 个信号？
1. **单位错误**：`min_net_buy = 500`（意图 500 万元），但数据单位是元，500 元的阈值几乎不过滤 → 大量低质量信号进入后续流程。
2. **但信号数反而很少**：因为阈值太低，几乎所有龙虎榜记录都通过了净买入过滤，但"5天内出现2次+"的条件仍然生效，加上仅在 `bear_low` 状态下启用（占 24.1% 交易日），最终只有 8 个信号。
3. **实质问题**：不是信号太少，而是过滤逻辑错位——该过滤的（低金额）没过滤，该放行的（高金额）因为其他条件被卡住。

### V3 修复后的表现
- 修正阈值为 `5,000,000`（500 万元），有效过滤掉散户主导的小额龙虎榜记录
- confidence 计算以亿元为基准归一化，分布合理（0.71 ~ 0.95）
- 574 个信号分布在 3 年中，平均每个交易日约 0.8 个事件信号
- events 表覆盖 2020-01-02 ~ 2026-04-09，共 96,137 条龙虎榜记录，数据充足

### 事件驱动策略的收益贡献
事件驱动策略对 V3 收益提升的贡献体现在：
- **平均盈利提升**：+11.98% → +14.15%（+2.17%），龙虎榜股票的短期爆发力强
- **盈亏比提升**：1.33 → 1.46，机构集中买入的股票安全边际较高
- **多策略协同**：multi 信号从 47 增至 65，事件信号与其他策略形成交叉验证

## 3.4 市场状态分布（三版本一致）

```
neutral_low: 420 天 (57.9%)  ← 震荡偏弱
bear_low:    175 天 (24.1%)  ← 熊市低波
bull_low:    125 天 (17.2%)  ← 牛市低波
neutral_high:  3 天 (0.4%)
bull_high:     3 天 (0.4%)
```

在 82% 的交易日（neutral_low + bear_low）中，当前策略权重配置为：
- Value: 30%（防守核心）
- Event: 10%（仅 bear_low）/ 0%（neutral_low）
- Momentum: 5%（大幅压缩）
- Growth: 5%（大幅压缩）

**待优化**：事件驱动策略当前仅在 `bear_low` 状态下启用（权重 10%），但龙虎榜事件本身与市场状态关联不大。考虑在 `neutral_low` 和 `bull_low` 状态下也分配一定权重，有望进一步提升信号数量和收益。
