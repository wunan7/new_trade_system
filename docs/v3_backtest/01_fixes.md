# 1. 本轮修复内容

## 1.1 舆情因子时序修复

### 问题描述
模拟盘每日 19:30 运行时，`stock_sentiment_daily` 当天数据尚未入库（通常在 22:00 左右生成），导致 `_load_sentiment_data()` 查询当天日期始终返回空结果，3 个舆情因子（sentiment_score、news_heat、news_mention_count）自系统上线以来从未生效。

### 修复方案
**文件**: `trading_system/factors/sentiment.py`

1. 查询目标从当天(T)改为 **T-1**（前一日），因为前一日的舆情数据在当天 19:30 运行时一定已入库。
2. 增加 **7 天回溯窗口**：如果 T-1 没有数据（周末、假期、采集中断），自动向前查找最近 7 天内可用的最新数据。
3. 仅保留最近一天的数据，避免跨日期混用。
4. 新增日志输出实际使用的舆情日期，方便排查。

### 验证结果
- 修复前：每天 `Sentiment factors computed for 0 stocks`
- 修复后：04-09 加载 04-08 数据（990 只），04-02 加载 04-01 数据（2418 只）

## 1.2 事件驱动策略单位修复

### 问题描述
`events` 表中 `content` 字段格式为 `"净买入: 5673691.00万元"`，但实际存储的数字**单位是元**而非万元（标注错误）。

策略代码 `EventDrivenStrategy` 中 `min_net_buy = 500`，原意是过滤净买入低于 500 万元的记录。但由于数据实际单位是元，阈值 500 元几乎不起过滤作用，同时 confidence 计算公式 `net_buy / 10000 * 0.05` 也会因为数值过大而溢出到上限 0.95。

**证据**：
- `000070` 的净买入值为 `377767547.74`
- 如果是万元 → 37,776 亿元（荒谬）
- 如果是元 → 3.78 亿元（合理，符合龙虎榜典型金额）

### 修复方案
**文件**: `trading_system/strategies/event_driven.py`

1. `min_net_buy`: `500` → `5_000_000`（5,000,000 元 = 500 万元）
2. `confidence` 计算: `net_buy / 10000 * 0.05` → `net_buy / 1e8 * 0.05`（以亿元为基准归一化）

### 验证结果
- 修复前：3 年回测仅产生 8 个事件驱动信号
- 修复后：单日（2026-04-09）产生 9 个信号，3 年回测产生 574 个信号
- 净买入金额范围合理：4,772 万 ~ 5.7 亿元
- confidence 分布合理：0.71 ~ 0.95
