# 模拟盘系统概述与架构

## 一、什么是模拟盘

模拟盘是一套**不投入真实资金**的交易仿真系统。它以 100 万虚拟本金为起点，每个交易日自动执行完整的选股→风控→下单→记账流程，用真实的 A 股行情数据驱动，但所有买卖都只记录在数据库中，不对接券商。

简单说：**用真行情假钱，跑真策略假单**。

## 二、为什么需要模拟盘

| 目的 | 说明 |
|------|------|
| **验证策略有效性** | 回测用的是历史数据（已知结果），模拟盘是面对未知的真实市场，是策略从"理论可行"到"实战可用"的必经环节 |
| **积累实盘经验** | 观察策略在不同市场环境（牛/熊/震荡）下的真实表现，建立对策略行为的直觉 |
| **暴露系统 Bug** | 回测是批量跑的，模拟盘是逐日跑的，能发现回测中不会出现的时序问题、数据延迟、边界条件等 |
| **建立绩效基线** | 积累足够多的模拟盘交易记录后，可以计算出策略的真实夏普比率、最大回撤、胜率等指标，为实盘决策提供依据 |
| **累积北向因子数据** | north_net_buy 当前无历史数据源，只能靠每日采集逐步累积，模拟盘运行 20 天后北向因子即可生效 |

│  3. 交易约束         │ ─────────> │  Step 3.  加载组合状态             │

## 五、核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始资金 | 100 万元 | 模拟本金 |
| 持仓数量 | 5-15 只 | 由市场状态和信号数量决定 |
| 单股上限 | 15% | 单只股票最大仓位占比 |
| 行业上限 | 30% | 同一申万一级行业最大占比 |
| 总仓位上限 | 20%-90% | 由市场状态决定（熊市20%，牛市90%） |
| 佣金 | 万 2.5 双边 | 最低 5 元/笔 |
| 印花税 | 千 1 | 仅卖出收取 |
| 滑点 | 万 2 | 模拟市场冲击成本 |
| 回撤熔断 | 18% | 净值从高点回撤超 18% 全部清仓 |

## 六、市场状态与仓位控制

系统通过沪深 300 指数自动判断当前市场处于 6 种状态之一，并据此调整策略权重和总仓位上限：

| 市场状态 | 判断条件 | 价值策略 | 成长策略 | 动量策略 | 总仓位上限 |
|---------|---------|---------|---------|---------|-----------|
| 牛市+低波 | 收盘>MA60, 波动率低 | 20% | 35% | 25% | **90%** |
| 牛市+高波 | 收盘>MA60, 波动率高 | 10% | 25% | 35% | **80%** |
| 震荡+低波 | 收盘≈MA60, 波动率低 | 30% | 15% | 10% | **60%** |
| 震荡+高波 | 收盘≈MA60, 波动率高 | 25% | 10% | 10% | **50%** |
| 熊市+低波 | 收盘<MA60, 波动率低 | 40% | 5% | 5% | **40%** |
| 熊市+高波 | 收盘<MA60, 波动率高 | 30% | 0% | 0% | **20%** |

**核心逻辑：熊市少买、牛市多买；高波动减仓、低波动加仓。**

## 七、三张核心数据表

### portfolio_positions（持仓记录）

每买入一只股票创建一条 `status='open'` 的记录，卖出时更新为 `status='closed'` 并记录盈亏。

| 关键字段 | 说明 |
|---------|------|
| code, open_date, open_price, shares | 什么时候买的、买了多少 |
| strategy_source | 来自哪个策略（value/growth/momentum） |
| stop_loss_price, take_profit_price | 止损止盈价位 |
| status | open=持有中, closed=已平仓 |
| close_reason | 为什么卖的（stop_loss/trailing_stop/time_limit） |
| pnl_pct | 盈亏百分比 |

### trade_log（交易流水）

每笔买卖操作的完整记录，含真实交易成本。

| 关键字段 | 说明 |
|---------|------|
| direction | BUY 或 SELL |
| price, shares, amount | 成交价、股数、金额 |
| commission, stamp_tax, slippage | 佣金、印花税、滑点 |

### portfolio_nav（每日净值）

每个交易日一行，记录组合总价值和绩效指标。

| 关键字段 | 说明 |
|---------|------|
| total_value, cash | 总资产、现金 |
| daily_return | 当日收益率 |
| cumulative_return | 累计收益率 |
| benchmark_return | 沪深300当日收益率（对标基准） |
| excess_return | 超额收益 = 策略收益 - 基准收益 |
