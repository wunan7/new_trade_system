# 历史估值数据（stock_valuation）补全需求说明

## 背景与问题描述
在执行 A 股智能交易决策系统（new_solution）的 3 年完整回测（2023-04 至 2026-04）时发现，回测前两年的策略分配存在严重失衡：
- **动量策略**（基于量价）触发了 2935 次买入（占比 84.6%）
- **价值策略**（基于低估值/高股息）仅触发了 13 次买入（占比 0.3%）

排查后确认，原因是底层 `finance` 数据库中的 `stock_valuation`（每日估值）表在 **2023 年至 2025 年中旬这段时间完全没有数据**。
由于价值策略的核心逻辑（PE 分位 < 30%、PB 分位 < 30%）强依赖历史估值数据，数据缺失导致该策略在回测前两年被强制失效，系统被迫单靠动量策略在熊市中运行，导致回测失真（总收益 -19.86%，最大回撤 21.84%）。

## 数据补全需求

为了能够准确评估多策略融合（价值 + 成长 + 动量 + 事件）在长周期（牛/熊/震荡市）中的真实表现，需要通过 `finance_data` 数据底座项目，将缺失的历史每日估值数据补齐。

### 1. 补全时间范围
- **起始日期**：2020年01月01日（或系统内 `stock_daily` 的起始日期）
- **结束日期**：2025年06月30日
- **检查更新**：同步检查一下`stock_valuation` 表中2025年06月30日之后的数据是否存在，如果有缺失，也请补全。

### 2. 目标数据表
数据库 `finance` 中的 `stock_valuation` 表。

### 3. 需要补全的核心字段
补全脚本需确保以下核心字段在历史每个交易日都有值（这是当前因子引擎计算所需的必需字段）：
- `code`（股票代码）
- `trade_date`（交易日期）
- `pe_ttm`（滚动市盈率）
- `pb`（市净率）
- `ps_ttm`（滚动市销率）
- `circulating_market_cap`（流通市值，用于计算资金流向强度占比）
- `total_market_cap`（总市值）

### 4. 实施建议
1. **数据源**：建议使用 Tushare Pro 的 `daily_basic`（每日指标）接口批量获取历史区间的上述字段，或使用 AKShare 的对应接口。
2. **入库方式**：按日或按月分批拉取后，批量插入/更新（UPSERT）到 `stock_valuation` 表中，注意处理复权和退市股票（避免生存者偏差）。
3. **验证**：补全完成后，确保 `SELECT COUNT(*) FROM stock_valuation WHERE trade_date BETWEEN '2023-04-01' AND '2023-04-30'` 能返回正常数量的记录（每日约 5000+ 条）。
4. **检查**：检查现在每日定时任务中是否都有正常获取`stock_valuation`表数据的逻辑和正确结果
