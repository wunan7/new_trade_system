### 4.2 IC/IR 评估关键发现（2026-04-11）

基于 43 个因子在 35 个月的评估结果：

**有效因子（IR ≥ 0.5，全部为反向因子）：**

| 因子 | IR | 方向 | 含义 |
|------|:--:|:----:|------|
| margin_chg_rate | -0.72 | 反向 | 融资增长 → 未来下跌（追涨情绪） |
| upper_shadow_ratio | -0.65 | 反向 | 上影线大 = 抛压重 |
| momentum_60d | -0.62 | 反向 | A股强烈反转效应 |
| turnover_dev | -0.60 | 反向 | 异常换手 → 散户追热点 |
| pb | -0.55 | 反向 | 低PB价值效应 |
| dividend_yield | **+0.50** | **正向** | 唯一有效正向因子，胜率74% |

**关键洞察**：
1. A股存在强烈的**反转效应**，而非动量效应——这与策略设计中的动量策略理论基础存在冲突
2. **传统基本面因子（ROE、净利润率、营收增长）几乎无效**（IR < 0.1），但设计中Growth策略重度依赖这些因子
3. 替代因子（市场拥挤度等）尚未评估，可能存在高IC的未发现因子

### 4.3 未被利用的已有数据

**TrendRadar `ai_analysis_results`** 已生产每日4次结构化分析，完全未接入因子系统：
- `sector_impacts_json` — 板块情绪评分（sector/impact/confidence）
- `risk_alerts` — 风险预警
- `macro_impact` — 宏观影响分析

## 五、Layer 3 策略层：差距详情

### 5.1 策略完成情况

| 策略 | 设计 | 实现 | 状态 |
|------|------|------|:----:|
| 价值策略 | 低估+质量+股息 | ✅ 基本实现 | 已完成 |
| 成长策略 | 高成长+ROE趋势+PEG | ✅ 基本实现（缺PEG历史优化） | 已完成 |
| 动量策略 | 强者恒强+量价确认 | ✅ 基本实现 | 已完成 |
| 事件驱动策略 | 多类型事件+LLM解析 | ⚠️ 仅龙虎榜，缺业绩超预期等 | 部分完成 |
| **技术择时策略** | 为其他策略提供入场时机 | **❌ 完全未实现** | 缺失 |

### 5.2 技术择时策略的战略意义

设计文档明确指出：
> 技术择时策略不独立选股，而是作为**其他4个策略的时机调整器**——当其他策略产生买入信号时，决定"现在买"还是"等更好的入场点"

**当前系统的问题**：价值策略发现低估值股票，但该股处于下跌趋势中 → 立即买入 → ATR止损触发 → 亏损。技术择时层可在此等待趋势反转再入场，直接提升胜率。

### 5.3 市场状态检测维度差距

| 维度 | 设计 | 当前 | 状态 |
|------|------|------|:----:|
| 趋势（MA关系/ADX） | ✅ | ✅ | 已实现 |
| 波动率 | ✅ | ✅ | 已实现 |
| 市场宽度（涨跌家数比） | ✅ | ❌ | 缺失 |
| 资金面（北向20日累计） | ✅ | ❌ | 历史数据太短 |
| 情绪（换手率分位/涨跌停比） | ✅ | ❌ | 缺失 |
| 宏观（PMI/CPI等） | ✅ | ❌ | 数据未采集 |

**结果**：当前6种市场状态的判断基于2个维度，准确性存在较大限制，导致策略权重配置经常不准确。

## 六、Layer 4 风控层：差距详情

### 6.1 已实现

- ✅ 单股仓位限制（≤15%）
- ✅ 单行业限制（≤30%）
- ✅ 流动性约束（日均成交>500万）
- ✅ ST/新股过滤
- ✅ Kelly公式仓位定量
- ✅ 固定止损 + ATR动态止损
- ✅ 移动止盈（三档递进）
- ✅ 四级回撤熔断（正常/黄色/橙色/红色）
- ✅ 时间止损

### 6.2 缺失

| 缺失项 | 设计描述 | 影响 |
|--------|---------|------|
| 相关性约束 | 持仓间相关系数<0.7（熊市）/<0.8（牛市） | 无法控制系统性风险集中 |
| 涨跌停约束 | 涨停板不可买入/跌停板不可卖出 | 回测存在不可交易信号 |
| 压力测试 | 2015崩盘/2018熊市/2020疫情场景 | 无法评估极端情况下的系统表现 |
| 策略归因（Brinson） | 超额收益分解为资产配置+选股+交互效应 | 无法定量诊断哪个策略在贡献超额 |
| 动态滑点模型 | 基于成交量的滑点（小盘股更大） | 当前固定万2滑点低估真实成本 |

## 七、Layer 5 LLM层：完全未实现

### 7.1 设计的3个Agent

**Agent 1：信号仲裁Agent**
- 触发条件：同一股票两策略权重合计>40%且方向相反；或宏观风险预警时个股信号仍正面
- 输出：仲裁逻辑 + 综合方向 + 置信度 + 监控条件
- **当前替代方案**：简单加权平均，无智能仲裁

**Agent 2：报告生成Agent**
- 为推荐股票生成可读研究：一句话结论、核心逻辑、关键数据、风险提示
- **当前替代方案**：无，只有数字信号

**Agent 3：风险预警Agent**
- 监控宏观事件（央行政策、国际事件）、市场异常（单日-3%、北向大幅流出）
- 接入TrendRadar的`risk_alerts`字段 + 持仓专属风险
- **当前替代方案**：仅有回撤熔断，无前瞻性预警

### 7.2 TrendRadar数据的巨大浪费

TrendRadar系统每天已在生产高质量AI分析，但**完全未被交易系统消费**：

| TrendRadar输出 | 更新频率 | 当前使用 | 潜在用途 |
|----------------|---------|---------|---------|
| `sector_impacts_json` | 每日4次 | ❌ 未使用 | 板块情绪因子、行业轮动信号 |
| `risk_alerts` | 每日4次 | ❌ 未使用 | 风险预警Agent输入 |
| `macro_impact` | 每日4次 | ❌ 未使用 | 宏观维度市场状态判断 |
| `trading_strategy` | 每日4次 | ❌ 未使用 | LLM仲裁参考 |
| `ai_filter_results` | 实时 | ❌ 未使用 | 个股新闻情绪因子 |
| TrendRadar MCP (29工具) | 实时 | ❌ 未使用 | Agent工具接口 |

## 九、从MVP到完整系统：优先级路线图

### 第一阶段：高杠杆改进（预期年化收益提升5-10%）

**1.1 接入TrendRadar → 板块情绪因子**
- 工作量：2-3天
- 从`ai_analysis_results.sector_impacts_json`提取板块情绪分
- 映射到个股（通过行业分类）
- 作为新因子加入factor_cache，评估IC/IR

**1.2 实现信号仲裁Agent（Layer 5第一步）**
- 工作量：3-5天
- 使用Claude API
- 仅在触发条件满足时调用（降低成本）
- 输出结构化JSON，替换当前简单加权平均

**1.3 扩充事件驱动：业绩超预期+管理层增持**
- 工作量：3-4天
- 采集公告数据（AKShare `stock_notice_report`）
- 修复insider_trades代码bug
- 在events表增加新事件类型

**1.4 技术择时策略**
- 工作量：3-5天
- 不独立选股，作为其他策略的入场时机过滤器
- MACD金叉、均线突破、布林带位置
- 可将当前动量策略的止损率显著降低

### 第二阶段：系统完善（预期年化收益提升3-5%）

**2.1 市场状态检测增强**
- 补充市场宽度维度（涨跌家数比）
- 补充情绪维度（换手率分位）
- 采集宏观数据 + 接入宏观维度

**2.2 风险预警Agent**
- 接入TrendRadar `risk_alerts`
- 持仓专属负面事件监控
- 盘前预警推送

**2.3 飞书/微信推送**
- 盘前7:30决策面板
- 盘后16:00组合盈亏
- 复用TrendRadar已有通知基础设施

### 第三阶段：长期完善

**3.1 报告生成Agent**：为每个推荐股票生成可读买入逻辑

**3.2 策略归因分析（Brinson模型）**：定量诊断超额收益来源

**3.3 Streamlit可视化面板**：因子有效性、组合表现、信号历史

**3.4 压力测试**：模拟2015/2018/2020极端场景
