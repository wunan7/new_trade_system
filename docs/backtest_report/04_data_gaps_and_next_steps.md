# 4. 核心数据缺失与下一步计划 (2026-04-05)

## 回测失真的根本原因

为什么在之前长达三年的历史中，“价值策略”（主打低估值、高股息的防守型标的）仅仅被触发了 13 次买入，而“动量策略”（追涨杀跌）主导了 84.6% 的交易？

核心原因在于：底层的 `finance` 数据库在 **2023 年至 2025 年中旬这段时间，完全缺失了 `stock_valuation`（每日估值）表的数据**。

价值策略的核心逻辑是筛选 `PE_TTM < 30% 分位` 且 `PB < 30% 分位` 的股票，而在这段回测时间里：
- `pe_ttm`：全空（0%）
- `pb`：全空（0%）
- `dividend_yield`：全空（0%）
- `ps_ttm`：直到最近依然因为上游未抓取而全空（0%）

因为系统没有得到任何历史市盈率（PE）或市净率（PB）的输入，因子引擎 `factor_cache` 在处理 2023 和 2024 年的数据时，估值相关列均存为 `NULL`。在最终策略生成层面，这些被筛选的标的一律判定为“不符合估值后30%分位条件”，导致原本设计上应该在 `bear_low` 和 `neutral_low` 时段承担主要持仓权重的“价值策略”**被强制静音了整整两年**！

因此，系统在这段长达两年的“熊市/震荡”时期里，完全单腿依赖于“动量策略”在操作，频繁被诱多打损，才导致了惨淡的 3 年回测成绩（-19.86%）。

## 短期表现（数据完整期）验证

为了验证系统在数据完整的周期内是否能展现出应有的能力，我们又进行了一次**近 9 个月（2025-06-30 ~ 2026-04-03）的回测**。
这一时期由于包含了较新的定时采集任务，数据是完整的（价值因子、资金面因子、舆情因子齐备）：
- **总收益率**：+4.14%（年化 +6.27%）
- **最大回撤**：仅 4.89%
- **动量与价值策略的比例**：策略分配更趋于理性，价值策略信号（27次）、事件驱动（5次）开始发力。
这证明在数据完整时，系统的多因子、多策略融合是有效的防守反击型框架。

## 下一步工作安排

**首要任务**：补全历史核心数据。
目前已输出 [历史估值数据补全需求](../valuation_data_backfill_requirement.md) 给数据采集组。一旦该组完成以下动作：
1. 从 `Tushare Pro` 等接口补齐 `2020-01-01` 至 `2025-06-30` 的 `stock_valuation` 数据。
2. 修复 `money_flow` 的资金流向上游缺失项。

**后续推进步骤**：
1. **重新运行全量因子回填**：`python backfill_factors.py` 更新全市场的价值和资金因子。
2. **开展全策略的真实回测**：届时再看这 3 年，在价值策略与动量策略合理对冲的情况下，回撤将被进一步压缩，同时系统有望在熊市捕捉到低位的高息蓝筹，在牛市利用动量和成长放大收益。
3. **参数敏感性分析**：基于完整回测跑出不同市场状态下的权重调整（目前熊市下可能对价值的 30% 分位切得还不够宽，需要根据全市场实际分布重新校准）。
4. **LLM 决策层**：目前 1-5 层已全部完成量化风控验证，一旦底层量化表现达到设计文档中“跑赢基准 20%”的要求，就可以接入最后的设计环节——LLM 信号仲裁与预警。
