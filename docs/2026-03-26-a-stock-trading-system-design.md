# A股智能交易决策系统设计文档（

## 一、项目概述

### 目标
构建一个半自动决策辅助系统，在中国A股市场实现年化超额收益20-50%（相对沪深300），最大回撤控制在20%以内，以1年为评估周期。

### 核心约束
- **自动化程度**：半自动——系统给出买卖信号和推荐，人类做最终决策并手动执行
- **资金规模**：小资金（<100万），流动性约束小，可灵活操作小盘股
- **技术栈**：Python为主
- **交易风格**：多风格自适应（短线/波段/中长线根据市场状态切换）
- **风险偏好**：最大回撤控制在20%以内
- **数据预算**：<500元/月
- **AI融合**：NLP + ML + LLM 全面融合

### 架构选型：方案A — 多因子Alpha引擎 + LLM决策层

以量化多因子模型为底层alpha发现引擎，上层用LLM做多源信息融合和决策推理。

**优势：**
- 因子模型可回测、可量化、可归因——知道赚的钱从哪来
- LLM层不直接预测价格（这很难），而是做信息整合和推理（这是LLM擅长的）
- 多策略自适应——不同市场环境激活不同策略组合
- 可渐进式构建：先上因子引擎，再加LLM层

## 二、系统总体架构（6层）

```
┌─────────────────────────────────────────────────────┐
│              第6层：人机交互层                         │
│    信号仪表盘 / 持仓看板 / 回测报告 / 推送通知          │
├─────────────────────────────────────────────────────┤
│              第5层：LLM决策融合层                      │
│    多源信息整合 / 信号冲突仲裁 / 自然语言研报生成        │
├─────────────────────────────────────────────────────┤
│              第4层：风控与仓位管理层                    │
│    止损止盈 / 仓位计算 / 相关性约束 / 回撤控制          │
├─────────────────────────────────────────────────────┤
│              第3层：多策略信号层                        │
│    ┌──────────┬──────────┬──────────┬──────────┐    │
│    │ 价值策略  │ 动量策略  │ 事件驱动  │ 技术择时  │    │
│    └──────────┴──────────┴──────────┴──────────┘    │
│    市场状态判断器 → 策略权重动态分配                     │
├─────────────────────────────────────────────────────┤
│              第2层：因子计算与特征工程层                 │
│    基本面因子 / 技术因子 / 另类因子 / ML特征             │
├─────────────────────────────────────────────────────┤
│              第1层：数据采集与存储层                     │
│    行情 / 财务 / 舆情 / 公告 / 宏观 / 另类数据           │
└─────────────────────────────────────────────────────┘
```

**核心设计原则：**
- **层间解耦**：每层通过标准化接口通信，可独立开发、测试、替换
- **信号可归因**：每个买卖信号都能追溯到具体因子和策略来源
- **LLM不预测价格**：LLM只做信息整合和推理，不直接预测涨跌
- **人在回路**：最终买卖决策由人做出，系统提供决策依据

**关键设计决策：**
- **复用现有基础设施**：`finance`库和`finance_public_opinion`库已运行稳定，交易系统直接复用，新增表放入`finance`库
- **主存储用PostgreSQL**：成熟稳定、SQL生态丰富、支持JSON字段存舆情等半结构化数据、支持分区表管理大量行情数据
- **Parquet做归档**：分钟级历史数据量大，用Parquet列式压缩存储，通过pandas直接读取
- **复用`finance_data`包**：现有查询API（`get_stock_prices`等）直接供因子计算层调用，避免重复开发
- **复用TrendRadar**：舆情因子直接查询`finance_public_opinion.ai_analysis_results`的结构化字段，无需额外LLM处理
- **增量更新**：每个数据表记录`last_update`/`updated_at`，采集器只拉取增量数据（现有采集器已实现此机制）
- **数据质量校验**：入库前检查缺失值、异常值、复权一致性，标记而非删除异常数据

### PostgreSQL核心数据表Schema

> **注意**：以下仅列出交易系统**需要新建**的表。已有表（`stock_daily`, `stock_valuation`, `financial_*`等）的完整Schema见 `finance_data/数据字典.md`。
>
> 已有表与设计需求的差异：
> - `stock_daily` 无独立 `adj_factor`（已是前复权数据）、无 `is_suspended`/`up_limit`/`down_limit` → 需扩展字段或新建辅助表
> - `stock_daily` 字段名用 `code` 而非 `stock_code` → 因子计算层做字段映射
> - `stock_valuation` 无 `PCF`（市现率） → 可由现金流表计算

```sql
-- 🆕 资金流向（日级更新）
CREATE TABLE money_flow (
    trade_date      DATE NOT NULL,
    stock_code      VARCHAR(10) NOT NULL,
    north_net_buy   DECIMAL(16,4),          -- 北向资金净买入（万元）
    north_hold_pct  DECIMAL(8,4),           -- 北向持仓占比(%)
    main_net_inflow DECIMAL(16,4),          -- 主力净流入（万元）
    big_order_ratio DECIMAL(8,4),           -- 大单净流入比(%)
    margin_balance  DECIMAL(16,4),          -- 融资余额（万元）
    margin_buy      DECIMAL(16,4),          -- 融资买入额（万元）
    source          VARCHAR(20),
    PRIMARY KEY (trade_date, stock_code)
) PARTITION BY RANGE (trade_date);

-- 🆕 公告事件
CREATE TABLE events (
    id              SERIAL PRIMARY KEY,

    event_date      DATE NOT NULL,
    event_type      VARCHAR(50) NOT NULL,  -- earnings_beat, buyback, insider_buy, etc.

CREATE INDEX idx_events_stock_date ON events(stock_code, event_date);

    roe             DECIMAL(8,4),

    pb              DECIMAL(10,4),
    ps_ttm          DECIMAL(10,4),

    revenue_growth  DECIMAL(8,4),
    profit_growth   DECIMAL(8,4),
    debt_ratio      DECIMAL(8,4),
    -- 技术因子
    momentum_5d     DECIMAL(8,4),
    momentum_20d    DECIMAL(8,4),
    momentum_60d    DECIMAL(8,4),
    volatility_20d  DECIMAL(8,4),
    volume_ratio    DECIMAL(8,4),

) PARTITION BY RANGE (trade_date);

-- 🆕 信号历史（记录所有生成的信号，不仅是被执行的）
CREATE TABLE signal_history (

CREATE TABLE macro_data (
```

## 四、第2层：因子计算与特征工程层

### 因子体系总览

```
因子体系
├── 基本面因子（约30个）
│   ├── 盈利类：ROE、ROA、毛利率、净利率、扣非净利增速
│   ├── 成长类：营收增速、归母净利增速、经营现金流增速
│   ├── 估值类：PE_TTM、PB、PS、PCF、PEG、股息率
│   ├── 质量类：资产负债率、流动比率、应收占比、商誉占比
│   └── 盈余质量：经营现金流/净利润、应计利润比率
│
├── 技术因子（约25个）
│   ├── 动量类：5日/20日/60日涨幅、相对强弱(RS)
│   ├── 波动类：ATR、波动率(20日/60日)、布林带宽度
│   ├── 量价类：量比、换手率偏离、OBV斜率、量价背离
│   ├── 趋势类：MACD、均线多空排列得分、ADX
│   └── 微观结构：涨停/跌停频率、振幅均值、上下影线比
│
├── 资金面因子（约10个）
│   ├── 北向资金：个股北向持仓变化、行业北向流入
│   ├── 主力资金：大单净流入比、主力资金连续流入天数
│   └── 杠杆类：融资余额变化率、融资买入占比
│
├── 舆情因子（约8个，由LLM解析生成）
│   ├── 新闻情感得分（正/负/中性加权）
│   ├── 研报一致预期变化方向
│   ├── 社交媒体热度指数
│   └── 事件冲击分类（利好/利空/中性 + 强度）
│
└── 另类因子（约7个）
    ├── 市场状态：市场拥挤度、行业轮动速度、赚钱效应
    ├── 衍生品信号：期权PCR、股指期货基差率
    └── 截面因子：行业内相对排名、市值分位数
```

### 因子加工流水线

```
原始数据 → 清洗对齐 → 因子计算 → 去极值 → 标准化 → 中性化(可选) → 缓存入库
```

| 环节 | 方法 | 目的 |
|------|------|------|
| 清洗对齐 | 前复权统一、缺失值标记、ST/停牌标记 | 保证数据一致性 |
| 因子计算 | 按公式或模型计算原始因子值 | 提取信息 |
| 去极值 | MAD(中位数绝对偏差)法，3倍阈值 | 消除极端值干扰 |
| 标准化 | z-score截面标准化 | 因子间可比 |
| 中性化 | 对市值、行业做回归取残差(可选) | 剥离市值/行业暴露 |
| 缓存入库 | 写入PostgreSQL factor_cache表 | 避免重复计算 |

### 因子评价机制

定期（月度）评估每个因子的有效性：

| 评价指标 | 含义 | 有效阈值 |
|---------|------|---------|
| IC均值 | 因子值与下期收益的截面相关性 | \|IC\| > 0.03 |
| IC_IR | IC均值/IC标准差，衡量IC稳定性 | IC_IR > 0.5 |
| 分组单调性 | 按因子分5组，收益是否单调递增/递减 | 单调性得分 > 0.7 |
| 换手率衰减 | 因子信号的持续性 | 半衰期 > 5天 |
| 因子间相关性 | 与已有因子的冗余度 | 与任何已有因子相关性 < 0.7 |

因子评价结果反馈到第3层，失效因子自动降权或剔除。

## 五、第3层：多策略信号层

### 策略矩阵

```
                   市场状态判断器
    牛市/熊市/震荡 × 高波动/低波动 → 6种状态组合
                       ↓ 权重分配
  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────┐
  │ 价值策略 │ │ 成长策略 │ │ 动量策略 │ │ 事件驱动 │ │技术择时│
  │ 中长线   │ │ 中长线  │ │ 波段    │ │ 短线    │ │短线/  │
  │ 月级调仓 │ │ 月级调仓│ │ 周级调仓│ │ 事件触发│ │波段   │
  └─────────┘ └─────────┘ └─────────┘ └─────────┘ └───────┘
       ↓            ↓          ↓           ↓          ↓
              标准化信号输出（-1到+1）
```

### 五大策略模块

**策略1：价值策略**
- 核心逻辑：寻找被市场低估的优质公司，等待价值回归
- 选股因子：PB分位<30% + ROE>12% + 股息率>2% + 资产负债率<60%
- 增强因子：自由现金流为正、应计利润比率低、商誉占比低
- 排序模型：多因子加权评分，权重由IC_IR动态调整
- 调仓频率：月度
- 适用市场：震荡市、熊市后期

**策略2：成长策略**
- 核心逻辑：捕捉盈利高速增长且趋势延续的公司
- 选股因子：归母净利增速>30% + 营收增速>20% + ROE趋势上升
- 增强因子：研报一致预期上调、北向资金持续流入
- 排序模型：PEG修正 + 成长持续性ML评分
- 调仓频率：月度/季报后调整
- 适用市场：牛市、结构性行情

**策略3：动量策略**
- 核心逻辑：强者恒强，买入近期表现最强的股票
- 选股因子：20日动量前20% + 60日动量前30% + 量价配合
- 过滤条件：排除短期涨幅过大(>50%)的过热股、排除ST
- 反转保护：动量拥挤度指标>阈值时自动降权
- 调仓频率：周度
- 适用市场：趋势行情

**策略4：事件驱动策略**
- 核心逻辑：捕捉特定事件带来的短期定价偏差
- 触发事件：业绩超预期、股东增持、股权激励、并购重组、行业政策利好
- 信号生成：LLM解析公告→事件分类→历史同类事件统计超额收益→信号评分
- 持仓周期：1-10天，按事件类型预设
- 适用市场：所有市场状态

**策略5：技术择时策略（择时修正器）**
- 核心逻辑：基于量价形态和技术指标，对其他策略的入场/出场时机进行优化调整
- 信号来源：MACD金叉死叉、均线突破、布林带突破、量价异动
- ML增强：训练分类模型识别高胜率的技术形态组合
- 角色定义：不独立选股，作为其他4个策略的**择时修正器**——当其他策略产生买入信号时，技术择时决定"现在买"还是"等更好的入场点"；权重表中的"技术"列含义是：该比例的仓位受技术择时信号调节入场节奏
- 适用市场：高波动市场效果最好

### 市场状态判断器

输入特征：
- 大盘技术面：上证指数MA20/MA60关系、ADX趋势强度
- 市场宽度：上涨家数占比、新高新低比
- 波动率体系：沪深300波动率20日/60日、VIX类指标
- 资金面信号：北向资金20日累计、融资余额变化
- 情绪指标：换手率分位、涨停跌停比

输出6种状态：

|          | 低波动   | 高波动   |
|----------|---------|---------|
| 牛市      | 稳健上涨 | 急涨行情 |
| 震荡      | 横盘整理 | 剧烈震荡 |
| 熊市      | 阴跌行情 | 急跌恐慌 |

**状态→策略权重映射：**

| 市场状态 | 价值 | 成长 | 动量 | 事件 | 技术 | 总仓位上限 |
|---------|------|------|------|------|------|-----------|
| 稳健上涨 | 20% | 35% | 25% | 10% | 10% | 90% |
| 急涨行情 | 10% | 25% | 35% | 15% | 15% | 80% |
| 横盘整理 | 30% | 15% | 10% | 30% | 15% | 60% |
| 剧烈震荡 | 25% | 10% | 10% | 30% | 25% | 50% |
| 阴跌行情 | 40% | 5%  | 5%  | 35% | 15% | 40% |
| 急跌恐慌 | 30% | 0%  | 0%  | 40% | 30% | 20% |

## 六、第4层：风控与仓位管理层

### 风控体系架构

```
策略信号输入
    ↓
事前风控（交易前）→ 仓位计算引擎 → 事中风控（持仓期间）→ 事后风控（归因复盘）
    ↓
最终交易指令（含仓位、止损价、目标价）
```

### 事前风控：硬性约束

| 约束项 | 规则 | 理由 |
|--------|------|------|
| 单票仓位上限 | ≤15%总资金 | 防止单股黑天鹅摧毁组合 |
| 单行业上限 | ≤30%总资金 | 防止行业系统性风险 |
| 持仓数量 | 5-15只 | 小资金过度分散反而拖累收益 |
| 相关性约束 | 动态阈值：震荡/熊市<0.7，趋势市可放宽至<0.8；使用残差相关性（剥离市场beta后） | 真正分散风险 |
| 流动性过滤 | 日均成交额>500万 | 确保能进出 |
| ST/风险警示 | 排除 | 规避退市风险 |
| 上市时间 | >60个交易日 | 新股因子失效 |
| 停牌/涨跌停 | 跳过当日不可交易标的 | 避免无效信号 |

### 仓位计算引擎

三级仓位控制：

- **第一级（总仓位）**：由市场状态决定。急跌恐慌=20%, 阴跌=40%, 震荡=50-60%, 上涨=80-90%
- **第二级（策略间分配）**：由市场状态→策略权重表决定
- **第三级（个股仓位）**：改良Kelly + 波动率调整

```
个股仓位 = min(
    Kelly仓位 × 0.5,           # 半Kelly，更保守
    信号强度 × 策略分配额度,     # 信号越强仓位越大
    波动率倒数权重,              # 波动大的股票仓位小
    单票上限15%                  # 硬性天花板
)

Kelly仓位 = (胜率 × 盈亏比 - (1 - 胜率)) / 盈亏比
# 胜率：该策略历史滚动胜率（近60个交易日）
# 盈亏比：平均盈利 / 平均亏损
```

### 事中风控：止损止盈体系

**个股级：**
- 固定止损：短线策略 -5%，波段策略 -8%，中长线策略 -12%
- 移动止盈：盈利>10%后启动回落5%止盈，>20%收紧至4%，>40%收紧至3%
- 时间止损：持仓超过建议周期仍未达目标，减仓或清仓

**组合级回撤控制（最大回撤目标<20%）：**
- 黄色预警（回撤>8%）：禁止新开仓，仅允许持仓管理
- 橙色预警（回撤>12%）：全组合减仓至50%以下
- 红色预警（回撤>16%）：全组合减仓至20%以下
- 熔断线（回撤>18%）：清仓观望，等待人工复盘重启

**波动率应急：**
- 当日大盘跌幅>3%或VIX类指标突破历史90分位，自动触发全组合检查，推送紧急通知

### 事后风控：归因与监控

| 监控维度 | 具体内容 | 预警条件 |
|---------|---------|---------|
| 策略归因 | 每个策略的绝对收益和超额收益 | 单策略连续3周负超额 |
| 因子暴露 | 当前组合在各风格因子上的暴露度 | 单因子暴露>2倍标准差 |
| 集中度监控 | 个股/行业/风格的集中度 | 突破约束阈值 |
| 信号衰减 | 各策略IC滚动均值趋势 | IC连续下滑超过3个月 |
| 交易成本 | 实际滑点与预估差异 | 实际成本>预估×1.5 |

## 七、第5层：LLM决策融合层

### 核心定位

LLM在本系统中的角色：
- 做：整合多策略信号、融合定量与定性信息、生成分析报告、识别量化模型盲区
- 不做：不预测股价、不替代因子模型选股、不做最终决策

### 三个Agent

**Agent 1：信号仲裁Agent**

当多策略信号矛盾时进行仲裁。

**信号冲突触发条件（满足任一即触发仲裁）：**
1. 组合权重>40%的两个策略对同一股票给出相反方向（一个>+0.3，另一个<-0.3）
2. 综合信号在弱信号区间（-0.15到+0.15）且策略间方差大（标准差>0.4）
3. 风险预警Agent发出宏观风险警告，但个股信号仍为正向

输入包括市场环境、股票信息、各策略信号、近期事件。输出：

```json
{
  "综合方向": 0.35,
  "置信度": 0.55,
  "主要矛盾": "短期技术面和外部政策压制 vs 中期业绩高增长确定性",
  "仲裁逻辑": "业绩超预期是硬逻辑，但不确定性压制短期情绪...",
  "建议操作": "观望等待",
  "触发条件": "若利空落地且跌幅<5%，或股价站上20日均线，可建半仓",
  "关键监控": ["政策进展", "20日均线得失", "北向资金是否回流"]
}
```

**Agent 2：研报生成Agent**

为推荐清单中的股票生成简明研报，包含：一句话结论、核心逻辑、关键数据、风险提示、操作建议、信号来源追溯。

**Agent 3：风险预警Agent**

独立监控宏观和系统性风险：
- 宏观政策类：央行政策变化、重大监管政策、国际事件
- 市场异常类：大盘单日跌>3%、北向连续大幅流出、波动率突破90分位
- 持仓相关类：持仓股重大利空、行业系统性负面、突然放量异动

触发后LLM快速评估影响，生成预警报告并推送。

> **与TrendRadar的协同**：TrendRadar的`ai_analysis_results`已包含`macro_impact`、`risk_alerts`、`sector_impacts_json`等结构化字段，风险预警Agent可直接查询`finance_public_opinion`库获取舆情层面的风险信号，仅需额外处理持仓相关的个股风险和量化指标触发的风险。TrendRadar内置的MCP Server（`get_market_impact_summary`、`get_sector_sentiment`等29个工具）可作为Agent的工具调用接口。

### Prompt工程关键原则

1. 结构化输入：减少幻觉，提高一致性
2. 约束输出格式：JSON输出，便于程序解析
3. 禁止数值预测：LLM只做定性判断和逻辑推理
4. 提供历史锚点：给出类似事件的历史市场反应作参考
5. 强制对立面思考：要求列出反对理由，防止确认偏误

### 回测引擎

**执行模型：** 事件驱动回测（逐日模拟，非向量化），准确反映真实交易约束（涨跌停不可交易、停牌处理、T+1限制）。

**交易成本模型：**
- 佣金：万2.5（双向）
- 印花税：千1（仅卖出）
- 滑点：成交额的万2（模拟市价单冲击）
- 合计单次往返成本约0.12%

**偏差防护：**
- 前视偏差：严格使用T日收盘后数据生成T+1信号，财报数据以实际披露日为准而非报告期
- 幸存者偏差：股票池包含已退市股票的历史数据
- 过拟合防护：滚动窗口外推回测（训练期2年/测试期6个月滚动），不做全局参数优化

**基准定义：** 沪深300全收益指数（含分红再投资）

**关键输出指标：**

| 指标 | 定义 |
|------|------|
| 年化收益率 | 复合年化增长率(CAGR) |
| 年化超额收益 | 策略CAGR - 基准CAGR |
| 最大回撤 | 净值从峰值到谷值的最大跌幅 |
| 夏普比率 | (年化收益-无风险利率) / 年化波动率 |
| Calmar比率 | 年化收益 / 最大回撤 |
| 胜率 | 盈利交易次数 / 总交易次数 |
| 盈亏比 | 平均盈利金额 / 平均亏损金额 |
| 换手率 | 月均双边换手 |

**回测场景：**
- 单策略回测：独立验证每个策略历史表现
- 组合回测：模拟多策略融合+风控完整流程
- 归因分析：收益来自哪些因子、策略、时段
- 压力测试：2015股灾、2018熊市、2020疫情等极端期表现
- 参数敏感性：关键参数变化对结果的影响

### 数据版本化与可复现性

每日自动快照因子值和策略信号到不可变日志中（signal_history表定义见Part1），确保：
- 任意历史日期的系统状态可完整重建
- 因子定义变更后可对比新旧版本的信号差异
- 实盘信号与回测信号可直接对比，发现偏差

### 模拟盘阶段（Paper Trading）

系统上线前须经过至少3个月的模拟盘验证：
- 全流程运行（数据→因子→信号→风控→LLM→推送），但不实际交易
- 每日记录假设执行的交易和组合净值
- 对比模拟盘表现与同期回测预期，偏差>5%需排查原因
- 模拟盘通过后方可转入实盘
