# 4. 系统现存问题与优化方向

## 4.1 当前回测结果的不足

尽管完整数据版本的回测表现有了质的飞跃，但仍存在以下问题：

### 问题一：年化收益率偏低（+4.22%）
- **目标**：设计文档要求年化超额收益 20-50%（相对沪深300）。
- **现状**：年化收益 +4.22%，超额收益仅 +4.07%，远未达标。
- **可能原因**：
  1. 仓位过于保守：82% 的交易日处于 `neutral_low` 或 `bear_low`，总仓位上限仅 50%，大量资金闲置。
  2. 因子数量不足：设计目标 ~80 个因子，当前仅实现 33 个（其中有效的约 25 个）。
  3. 止损过于频繁：平均亏损 -6.64%，但部分股票可能只是短期回调就被止损出局。

### 问题二：夏普比率偏低（0.21）
- **目标**：一般认为夏普比率 > 1.0 才算优秀。
- **现状**：0.21，说明承担的风险没有得到足够的收益补偿。
- **可能原因**：
  1. 动量策略在震荡市中的频繁止损拉低了整体收益。
  2. 波动率 10.74% 偏高，但收益不足以覆盖。

### 问题三：事件驱动策略信号过少（8 次）
- **原因**：
  1. 龙虎榜数据仅从 2026 年 3 月底开始采集，历史数据缺失。
  2. 筛选条件过于严格（净买入 > 500 万 + 连续上榜 2 次）。
  3. 仅在 `bear_low` 状态下启用。
- **影响**：事件驱动策略几乎没有对组合收益产生贡献。

### 问题四：部分股票 entry_price 为 0
- 回测中有少量股票（如 600150、000878、688313 等）的入场价格为 0。
- 原因：这些股票在信号生成当天处于停牌状态，`stock_daily` 表中没有收盘价。
- 影响：这些信号被跳过，可能错过了复牌后的交易机会。

### 问题五：ps_ttm 因子始终为空
- `stock_valuation` 表中 `ps_ttm` 字段全为 NULL（AKShare 未采集此字段）。
- 影响：13 个基本面因子中有 1 个始终缺失，但对整体影响较小。

### 问题六：north_hold_pct 历史数据不足
- `north_hold_pct` 仅从 2026-03-28 开始有数据。
- 影响：2023-2026 年回测中，`north_flow_chg` 和 `north_days` 两个资金面因子在绝大部分时间为空。
- 这意味着市场状态检测器的"北向资金"维度在历史回测中实际上没有发挥作用。

## 4.2 短期优化方向（1-2 周）

### 优化一：仓位管理调优
**问题**：在 `neutral_low` 状态下总仓位上限仅 50%，导致大量资金闲置。
**建议**：
- 将 `neutral_low` 的仓位上限从 50% 提升至 60-70%。
- 在价值策略信号充足时，允许更高的仓位集中度。
- 增加"仓位利用率"指标到回测报告中，监控资金使用效率。

### 优化二：止损策略精细化
**问题**：当前使用统一的 ATR 动态止损 + 固定止损（-8% 或 -10%），对不同策略一视同仁。
**建议**：
- 价值策略：放宽止损至 -12%（低估值股票波动大但安全边际高）。
- 动量策略：收紧止损至 -6%（追涨策略需要快速止损）。
- 事件驱动：使用时间止损（持有超过 10 天未盈利则卖出）。

### 优化三：事件驱动策略增强
**问题**：龙虎榜数据历史缺失，且筛选条件过严。
**建议**：
- 补采 2023-2025 年的龙虎榜历史数据（AKShare 的 `stock_lhb_detail_daily_em` 接口）。
- 放宽筛选条件：净买入 > 300 万 或 连续上榜 1 次。
- 在 `neutral_low` 状态下也启用事件驱动策略（当前仅 `bear_low`）。

### 优化四：因子有效性评估（IC/IR）
**问题**：当前 33 个因子中，哪些真正有效、哪些是噪音，缺乏量化评估。
**建议**：
- 实现月度因子 IC（Information Coefficient）计算：每个因子值与下月收益率的 Rank IC。
- 计算 IR（Information Ratio）= IC 均值 / IC 标准差，筛选出 IR > 0.5 的有效因子。
- 根据 IC/IR 结果动态调整因子权重，淘汰无效因子。

## 4.3 中期优化方向（1-3 个月）

### 优化五：补充缺失因子
当前 33 个因子 vs 设计目标 ~80 个，缺失的高价值因子包括：
- **基本面**：ROA、PEG、流动比率、自由现金流（FCF）、扣非净利润增速
- **技术面**：KDJ、RSI、BOLL 通道宽度
- **资金面**：大宗交易折溢价率、融券余额变化率
- **另类因子**：行业轮动速度、个股拥挤度

### 优化六：回测引擎增强
- **滑点模型优化**：当前使用固定万 2 滑点，应改为基于成交量的动态滑点（小盘股滑点更大）。
- **涨跌停处理**：当前可能在涨停板买入或跌停板卖出，需要增加涨跌停不可交易的约束。
- **T+1 限制**：确保当天买入的股票不能当天卖出。

### 优化七：策略归因分析
- 实现 Brinson 归因模型，将超额收益分解为：
  - 资产配置效应（市场状态判断的贡献）
  - 选股效应（各策略选股能力的贡献）
  - 交互效应
- 这将帮助我们理解收益到底来自"选对了市场状态"还是"选对了股票"。

## 4.4 长期优化方向（3-6 个月）

### 优化八：LLM 决策融合层
设计文档中的第 5 层——LLM 决策融合层，目前尚未实现。
- **功能**：多源信息整合、信号冲突仲裁、自然语言研报生成。
- **前提**：底层量化策略的表现需要先达到设计目标（年化超额 20%+），才有必要引入 LLM 层。
- **实现路径**：先用 Claude API 做信号冲突仲裁的 PoC，验证 LLM 能否提升决策质量。

### 优化九：推送通知系统
- 实现盘前推送（今日决策面板）和盘后推送（今日组合盈亏）。
- 支持飞书/微信/邮件等多渠道推送。
- 可复用 TrendRadar 项目已有的通知基础设施。

### 优化十：模拟盘长期验证
- 系统已在 2026-03-27 开始模拟盘运行，需要至少 3 个月的模拟盘数据来验证：
  - 实盘信号与回测信号的一致性（偏差 < 5%）。
  - 实际交易成本与模型假设的差异。
  - 系统在真实市场环境中的稳定性。
